# Work tracking

Actionable work is tracked in
[GitHub Issues](https://github.com/joeblau/blau/issues). This file intentionally
does not duplicate issue bodies, estimates, or source line numbers because
those copies become stale.

Before starting work, use the issue list and its acceptance criteria as the
source of truth. New follow-ups should be filed with reproduction details,
platform/toolchain versions, expected behavior, and a bounded completion test.

The April 2026 simulator notes previously kept here were removed during the
documentation audit: their file paths and CI assumptions no longer described
the repository, and unscheduled proposals should not masquerade as committed
work. If multi-touch simulation, dropped-app installation, or model extraction
is prioritized again, open a focused issue against the current implementation.
